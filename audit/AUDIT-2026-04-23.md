# Greek.fi Smart Contract Security Audit — 2026-04-23

**Scope**: `foundry/contracts/{Factory,Option,Collateral,OptionUtils,YieldVault}.sol`, oracles, libraries, interfaces.
**Methodology**: Three parallel reviews — (1) core invariants + reentrancy + access control, (2) math / decimals / oracles, (3) tokens / Permit2 / vault. Each finding includes location, scenario, and remediation.

Detailed findings per surface live in:
- `audit/findings-core.md`
- `audit/findings-math.md`
- `audit/findings-integration.md`

This doc is the consolidated, severity-ranked executive summary. Duplicate findings across tracks are merged here and cross-referenced.

---

## Top-line risk picture

The protocol is internally well-structured (ReentrancyGuardTransient everywhere, `via_ir` optimization, 1:1 collateral invariant, clones with disabled initializers) but it has **five systemic design gaps**:

1. **Oracle trust is too permissive.** `_deployOracle` accepts any address returning the right `expiration()`; UniV3Oracle has no fallback path if the pool's observation buffer rolls off; the doc-sketched Chainlink path is unsafe.
2. **Factory allowance + operator approvals are globally scoped** across all clones and all future clones. Combined with `Option.mint(account, amount)` being callable by any third party, this is the single biggest multiplier on other vulnerabilities.
3. **YieldVault's `execute(target, calldata)` has no allowlist** — operator compromise = vault drain with no protocol guardrail.
4. **Weird-ERC20 tolerance is uneven.** FOT is caught at mint only, not at exercise, and not for rebasing / pausable / hook-enabled / blacklistable tokens.
5. **Post-expiry flows can permanently lock funds** via oracle failure, owner renounce, or blacklisted contract address.

Every Critical / High finding below is an instance of one of these five classes.

## Priority remediation order

Ordered by **impact × realism × remediation cost**:

1. **Gate third-party redemption / sweep authority.** (C-2, M-6, H-10-adjacent)
   Require `msg.sender == account || approvedOperator` on `redeem(account,…)` and `sweep(address)`. `sweep(holders[])` ok as keeper path but add grace period.
2. **Fix Factory allowance model.** (H-3, H-10, H-11)
   Scope `_allowances` to `(token, owner, option)`, OR gate `Option.mint(account,…)` on `msg.sender == account || approvedOperator(account, msg.sender)`. Single change closes four High findings.
3. **Add UniV3Oracle cardinality bump + emergency escape path.** (C-1, H-8)
   `POOL.increaseObservationCardinalityNext(…)` at construction, plus a fallback redeem path after a grace window past expiry.
4. **Lock down YieldVault execute / approvals.** (C-3, H-13)
   Target+selector allowlist on `execute`; domain-bound EIP-1271 envelope; operator/owner role split; reject `option == asset()` in `addOption`.
5. **Override `Factory.renounceOwnership`** to revert (H-4). Switch to `Ownable2Step`. Do same to `Collateral` (L-1).
6. **Fix UniV3Oracle tick math.** (H-7) `mulDiv` on inverse + explicit zero-guard + clamp near extremes.
7. **Fix `_redeemProRata` consideration leg.** (H-9, M-2) Pure pro-rata against actual consideration balance, not strike-rate. Unifies with `_redeemSettled`.
8. **Restrict oracle source.** (H-5, M-10) Factory-curated registry or enforce factory-deployed-only.
9. **Token compatibility matrix.** (H-2, H-12, M-7, M-11)
   - Blocklist ERC-777 / ERC-1363 via ERC-1820 registry sniffing.
   - Reject rebasing tokens (require wrapped/shares variants).
   - Bound `decimals() ≤ 36`.
   - Strict-equality FOT check on exercise.
10. **Chainlink branch pre-work** (H-6, L-3) — add staleness / round-completeness / positive-price / decimals checks when it ships.

## What the test suite misses

The Forge test suite is thorough on happy paths but missed entirely:
- Hook-enabled, rebasing, pausable, blacklistable ERC-20s.
- Adversarial `oracleSource` at `createOption` time.
- Third-party `redeem(victim,…)` and `sweep(victim)` griefing.
- Auto-mint combined with a hostile operator approval.
- UniV3 observation buffer rollover (simulate with `increaseObservationCardinalityNext(1)` then swap spam).
- Factory owner renounce/lock scenarios.

Recommend invariant-style fuzz tests against adversarial `oracleSource`, adversarial collateral/consideration tokens (FOT variants, rebasing mocks, ERC-777 mock, pausable mock), and adversarial third-party callers.

## Positive observations

- `Math.mulDiv` with `Rounding.Ceil` is used correctly wherever applicable.
- Clones deploy + initialize atomically (no front-run hijack window).
- `_disableInitializers()` on templates prevents takeover.
- `Option.renounceOwnership` is intentionally disabled — anti-rug move.
- `autoMintRedeem` is opt-in on **both** sender and receiver.
- `safeTransfer` / `forceApprove` used consistently; no raw `transfer`.
- `ReentrancyGuardTransient` (EIP-1153) is the modern, correct pattern.
- 1:1 collateral backing invariant holds in all non-adversarial paths checked.

The protocol has a sound core; most findings are about broadening the trust boundary beyond benign participants.
